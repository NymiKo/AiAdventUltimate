# FAQ - Часто задаваемые вопросы

## Настройка и установка

### Почему не работает авторизация?

Если у вас проблемы с авторизацией, проверьте следующее:

1. **Проверьте API ключ**: Убедитесь, что ваш API ключ DeepSeek правильно настроен в настройках приложения
2. **Проверьте интернет-соединение**: Приложение требует активного интернет-соединения для работы с API
3. **Проверьте формат ключа**: API ключ должен быть в формате строки без пробелов
4. **Проверьте баланс**: Убедитесь, что на вашем аккаунте DeepSeek есть доступные средства

### Как настроить API ключ?

1. Откройте настройки приложения (иконка шестеренки)
2. Найдите поле "DeepSeek API Key"
3. Введите ваш API ключ
4. Сохраните настройки

### Где получить API ключ DeepSeek?

1. Зарегистрируйтесь на [platform.deepseek.com](https://platform.deepseek.com/)
2. Перейдите в раздел API Keys
3. Создайте новый ключ
4. Скопируйте ключ и вставьте в настройки приложения

### Как настроить LM Studio для RAG?

1. Установите LM Studio с официального сайта
2. Запустите LM Studio и загрузите модель для эмбеддингов
3. Запустите локальный сервер на порту 1234
4. В настройках приложения укажите URL: `http://localhost:1234`
5. Убедитесь, что сервер запущен перед использованием RAG

### Нужен ли LM Studio для работы приложения?

Нет, LM Studio нужен только для работы RAG. Базовый чат с AI работает без LM Studio, используя только DeepSeek API.

## RAG (Retrieval-Augmented Generation)

### Что такое RAG?

RAG (Retrieval-Augmented Generation) - это технология, которая позволяет AI использовать информацию из вашей базы знаний для более точных ответов. Вместо того чтобы полагаться только на общие знания модели, AI ищет релевантную информацию в ваших документах.

### Как использовать RAG?

1. Убедитесь, что LM Studio запущен на localhost:1234
2. Перейдите в "Индекс эмбеддингов" (иконка хранилища)
3. Загрузите документы (HTML, PDF файлы)
4. Дождитесь обработки файлов
5. Включите RAG в настройках
6. Теперь AI будет использовать вашу документацию при ответах

### Почему RAG не работает?

- Убедитесь, что LMStudio запущен на localhost:1234
- Проверьте, что индекс эмбеддингов создан и содержит данные
- Убедитесь, что RAG включен в настройках
- Проверьте подключение к LM Studio в настройках
- Убедитесь, что модель для эмбеддингов загружена в LM Studio

### Какие форматы файлов поддерживаются для RAG?

Поддерживаются следующие форматы:
- HTML файлы (.html, .htm)
- PDF файлы (.pdf)
- Markdown файлы (.md, .markdown)
- Текстовые файлы (.txt)

### Сколько документов можно загрузить?

Технических ограничений нет, но рекомендуется загружать не более 100-200 документов за раз для оптимальной производительности. Большие объемы данных могут замедлить обработку.

### Как обновить индекс эмбеддингов?

Просто загрузите новые файлы через экран "Индекс эмбеддингов". Новые документы будут добавлены к существующему индексу.

### Можно ли удалить документы из индекса?

Да, вы можете очистить весь индекс через кнопку "Очистить" в экране эмбеддингов, а затем загрузить только нужные документы заново.

### Как улучшить качество ответов RAG?

1. Используйте качественные, структурированные документы
2. Убедитесь, что документы содержат релевантную информацию
3. Настройте параметры RAG в настройках (topK, rerankMinScore)
4. Используйте reranking для улучшения качества поиска

## Агенты

### Как создать агента?

1. Откройте боковую панель (иконка человека)
2. Нажмите кнопку "+" для создания нового агента
3. Заполните информацию об агенте:
   - Имя агента
   - Описание
   - Системный промпт (опционально)
4. Сохраните агента

### Что такое системный промпт агента?

Системный промпт - это инструкции для AI, которые определяют поведение агента. Например, вы можете создать агента-программиста с промптом "Ты опытный программист, специализирующийся на Kotlin и Android разработке".

### Как использовать нескольких агентов?

1. Выберите несколько агентов в боковой панели (отметьте их галочками)
2. Включите координатор для автоматического распределения задач
3. Задайте вопрос - координатор выберет подходящего агента или распределит задачу между несколькими

### Что такое координатор?

Координатор - это специальный агент, который управляет работой других агентов и распределяет задачи между ними. При включенном координаторе он автоматически выбирает наиболее подходящего агента для каждого вопроса.

### Можно ли работать без координатора?

Да, вы можете выбрать агентов вручную и работать с ними напрямую. Координатор - это опциональная функция для автоматизации.

### Как создать связь между агентами?

1. Откройте боковую панель → вкладка "Агенты"
2. Нажмите на иконку "Создать связь" (иконка Share)
3. Выберите исходного и целевого агента
4. Выберите тип связи (REVIEW, VALIDATE, ENHANCE, COLLABORATE)
5. Сохраните связь

### Что означают типы связей между агентами?

- **REVIEW**: Агент проверяет работу другого агента
- **VALIDATE**: Агент валидирует результаты другого агента
- **ENHANCE**: Агент улучшает результаты другого агента
- **COLLABORATE**: Агенты работают совместно над задачей

### Как удалить агента?

1. Откройте боковую панель → вкладка "Агенты"
2. Найдите нужного агента
3. Нажмите на иконку удаления
4. Подтвердите удаление

### Можно ли редактировать существующего агента?

Да, вы можете изменить имя, описание и системный промпт агента в любое время через интерфейс управления агентами.

## MCP Серверы

### Что такое MCP?

MCP (Model Context Protocol) - это протокол для интеграции внешних сервисов с AI ассистентом. Он позволяет AI использовать инструменты и данные из внешних систем.

### Как добавить MCP сервер?

1. Откройте боковую панель
2. Перейдите на вкладку "MCP Серверы"
3. Нажмите "+" для добавления нового сервера
4. Заполните конфигурацию:
   - ID сервера
   - Название
   - Команда запуска
   - Параметры (если нужны)
5. Сохраните сервер

### Какие MCP серверы поддерживаются?

Приложение поддерживает любые MCP серверы, совместимые с протоколом MCP. По умолчанию доступна интеграция с GitHub для работы с репозиториями.

### Как настроить GitHub MCP сервер?

1. Получите GitHub Personal Access Token
2. Добавьте MCP сервер GitHub через интерфейс
3. Укажите токен в параметрах сервера
4. Включите сервер

### Как проверить, работает ли MCP сервер?

В боковой панели на вкладке "MCP Серверы" вы увидите статус каждого сервера. Зеленый индикатор означает, что сервер работает.

### Почему MCP сервер не работает?

- Проверьте, что сервер включен
- Убедитесь, что команда запуска правильная
- Проверьте параметры сервера
- Убедитесь, что все необходимые зависимости установлены

## Голосовой ввод/вывод

### Как включить голосовой ввод?

Голосовой ввод включается автоматически на Desktop платформах, если ваша система поддерживает его. Используйте иконку микрофона для начала записи.

### На каких платформах работает голосовой ввод?

Голосовой ввод работает только на Desktop (JVM) платформах. На Android и iOS эта функция пока не поддерживается.

### Как использовать голосовой ввод?

1. Убедитесь, что микрофон подключен и работает
2. Нажмите на иконку микрофона в поле ввода
3. Говорите в микрофон
4. Нажмите красную кнопку для остановки записи
5. Текст автоматически появится в поле ввода

### Как включить голосовой вывод?

1. Откройте настройки приложения
2. Найдите опцию "Голосовой вывод"
3. Включите переключатель
4. Ответы AI будут автоматически озвучиваться

### Как настроить голосовой вывод?

Для работы голосового вывода требуется настройка Yandex Cloud API:
1. Создайте аккаунт в Yandex Cloud
2. Создайте сервисный аккаунт с ролью `ai.speechkit-tts.user`
3. Получите API ключ и Folder ID
4. Добавьте их в файл `.env` или настройки приложения

### Почему голосовой вывод не работает?

- Убедитесь, что голосовой вывод включен в настройках
- Проверьте, что API ключи Yandex Cloud настроены правильно
- Убедитесь, что у сервисного аккаунта есть нужные права
- Проверьте интернет-соединение

### Можно ли изменить голос для озвучивания?

В текущей версии используется голос Джарвиса через Yandex TTS. Изменение голоса пока не поддерживается.

## Проекты

### Как открыть проект?

1. Нажмите на название проекта в верхней панели (или "AiAdvent Ultimate", если проект не открыт)
2. Выберите "Открыть проект"
3. Укажите путь к папке проекта
4. Проект откроется автоматически

### Как закрыть проект?

1. Нажмите на название проекта в верхней панели
2. Выберите "Закрыть проект"

### Как работает интеграция с GitHub?

При открытии проекта, подключенного к GitHub:
- Приложение автоматически определяет репозиторий
- Показывается текущая ветка в верхней панели
- Доступны Pull Requests через боковую панель
- AI может использовать информацию о проекте в контексте

### Как просмотреть Pull Requests?

1. Откройте боковую панель
2. Перейдите на вкладку "Pull Requests"
3. Вы увидите список открытых PR
4. Можете запросить автоматический review через кнопку "Review"

### Как работает автоматический review PR?

1. Откройте вкладку "Pull Requests" в боковой панели
2. Найдите нужный PR
3. Нажмите кнопку "Review"
4. AI проанализирует изменения и добавит комментарий с review

### Почему не вижу информацию о ветке?

- Убедитесь, что проект открыт
- Проверьте, что проект подключен к Git репозиторию
- Убедитесь, что GitHub MCP сервер настроен и включен

## Чаты и сообщения

### Как очистить чат?

Нажмите иконку корзины в верхней панели чата. Это очистит текущий чат, но не удалит его из истории.

### Как сохранить чат?

Чаты сохраняются автоматически. Вы можете просмотреть список чатов, нажав на иконку чата в верхней панели.

### Как открыть старый чат?

1. Нажмите на иконку чата в верхней панели
2. Выберите нужный чат из списка
3. Чат откроется с полной историей сообщений

### Как создать новый чат?

1. Нажмите на иконку чата в верхней панели
2. Нажмите кнопку "Создать новый чат"
3. Или просто начните новый разговор - чат создастся автоматически

### Как удалить чат?

1. Откройте список чатов
2. Найдите нужный чат
3. Используйте опцию удаления (если доступна)

### Можно ли экспортировать чат?

В текущей версии экспорт чатов не поддерживается. История сохраняется локально в приложении.

## Настройки и производительность

### Приложение работает медленно

- Проверьте интернет-соединение
- Убедитесь, что LMStudio запущен локально (для RAG)
- Закройте ненужные чаты
- Проверьте, не слишком ли много агентов выбрано одновременно
- Уменьшите количество документов в RAG индексе

### Как оптимизировать работу RAG?

1. Используйте качественные, структурированные документы
2. Настройте параметры topK и rerankMinScore
3. Не загружайте слишком много документов одновременно
4. Используйте reranking для улучшения качества

### Как изменить тему приложения?

1. Откройте настройки
2. Найдите опцию "Темная тема"
3. Переключите переключатель

### Можно ли настроить параметры AI модели?

Да, в настройках доступны параметры:
- **Temperature**: Контролирует случайность ответов (0.0-1.0)
- **Max Tokens**: Максимальная длина ответа

### Что такое temperature?

Temperature - это параметр, который контролирует случайность ответов AI:
- Низкие значения (0.1-0.3): более детерминированные, точные ответы
- Высокие значения (0.7-1.0): более творческие, разнообразные ответы

### Что такое maxTokens?

MaxTokens - это максимальное количество токенов (слов/символов) в ответе AI. Большие значения позволяют получать более длинные ответы, но увеличивают время генерации и стоимость.

## Поддержка и тикеты

### Как получить помощь?

1. Используйте вкладку "Поддержка" в боковой панели
2. Задайте вопрос - AI найдет ответ в документации
3. Если проблема не решена, создайте тикет в системе поддержки

### Как работает система поддержки?

Система поддержки использует RAG для поиска ответов в документации и FAQ. Она также учитывает похожие тикеты других пользователей для более точных ответов.

### Как загрузить документацию для поддержки?

1. Откройте боковую панель → вкладка "Поддержка"
2. Переключитесь на "Документация"
3. Загрузите файлы с документацией (HTML, PDF, Markdown, TXT)
4. Документация будет использоваться для ответов в поддержке

### В чем разница между основным RAG и RAG поддержки?

- **Основной RAG**: Используется в основном чате для работы с кодом и проектами
- **RAG поддержки**: Используется только в поддержке для ответов на вопросы пользователей о приложении

## Ошибки и проблемы

### Приложение не запускается

- Проверьте, что установлена правильная версия Java (JDK 11+)
- Убедитесь, что все зависимости установлены
- Проверьте логи приложения на наличие ошибок

### Ошибка "API ключ не настроен"

1. Откройте настройки
2. Проверьте, что API ключ DeepSeek введен правильно
3. Убедитесь, что ключ не содержит пробелов
4. Сохраните настройки и перезапустите приложение

### Ошибка подключения к LM Studio

- Убедитесь, что LM Studio запущен
- Проверьте, что сервер работает на порту 1234
- Проверьте URL в настройках: должен быть `http://localhost:1234`
- Убедитесь, что модель загружена в LM Studio

### Ошибка при загрузке документов

- Проверьте формат файлов (поддерживаются HTML, PDF, Markdown, TXT)
- Убедитесь, что файлы не повреждены
- Проверьте размер файлов (слишком большие файлы могут вызвать проблемы)
- Убедитесь, что LM Studio запущен и доступен

### AI не отвечает или отвечает медленно

- Проверьте интернет-соединение
- Убедитесь, что API ключ действителен и есть баланс
- Проверьте, не слишком ли длинный запрос
- Попробуйте уменьшить maxTokens в настройках

### Голосовой ввод не работает

- Убедитесь, что вы используете Desktop версию
- Проверьте, что микрофон подключен и работает
- Проверьте права доступа к микрофону в системе
- Убедитесь, что Yandex API ключи настроены правильно

## Общие вопросы

### На каких платформах работает приложение?

Приложение работает на:
- **Android**: Полная поддержка всех функций кроме голосового ввода/вывода
- **iOS**: Полная поддержка всех функций кроме голосового ввода/вывода
- **Desktop (JVM)**: Полная поддержка всех функций, включая голосовые

### Можно ли использовать приложение офлайн?

Частично. Базовый чат требует интернет для работы с DeepSeek API. RAG может работать локально с LM Studio, но для загрузки документов и генерации эмбеддингов нужен интернет.

### Безопасно ли хранить API ключи в приложении?

API ключи хранятся локально на вашем устройстве. Они не передаются третьим лицам, кроме соответствующих сервисов (DeepSeek, Yandex Cloud) для работы функций приложения.

### Можно ли использовать несколько API ключей?

В текущей версии можно использовать только один API ключ DeepSeek за раз. Для переключения между ключами нужно изменить настройки.

### Как обновить приложение?

Обновление зависит от платформы:
- **Android**: Через Google Play или установку новой APK
- **iOS**: Через App Store
- **Desktop**: Скачайте новую версию и установите поверх старой

### Где найти логи приложения?

Логи выводятся в консоль при запуске приложения. На Desktop вы можете видеть их в терминале. На Android используйте logcat.

### Можно ли использовать другие AI модели кроме DeepSeek?

В текущей версии основная поддержка - DeepSeek. Yandex GPT доступен как альтернатива, но с ограниченной функциональностью.
